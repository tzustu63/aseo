## 🎓 使用時機對照

### 🔧 何時用「修復」

```
✅ 適用情境：
- 網站缺少基本 meta 標籤
- HTML 結構有明顯錯誤
- 需要快速修復技術問題

範例：
• 缺少 viewport
• 缺少 charset
• 缺少 canonical
```

### 🤖 何時用「AI 優化」

```
✅ 適用情境：
- Meta description 需要改寫
- Title 需要更吸引人
- 圖片需要有意義的 alt

範例：
• Meta 太短：85 字元 → AI 產生 154 字元
• Title 太長：80 字元 → AI 優化為 58 字元
• 圖片無 alt → AI 產生描述性文字
```

### ⚠️ 何時用「進階 AI 優化」

```
✅ 適用情境：
- H1 需要包含關鍵字
- 內容關鍵字密度不足
- 標題結構混亂
- 段落需要重寫

⚠️ 注意：
- 這會改變頁面核心內容
- 請先在測試環境測試
- 仔細閱讀 AI 分析
- 確認符合品牌風格

範例：
• H1 太簡單 → AI 產生 SEO 友好的標題
• 密度 0.3% → AI 重寫達到 1.5%
• H1→H4→H2 → AI 修正為 H1→H2→H3
```

### ℹ️ 何時用「人工處理」

```
⚠️ 需手動處理：
- 非常複雜的結構問題
- 品牌特定內容
- 法律或醫療內容
- 客戶評價和引用

建議：
查看工具建議後手動在 WordPress 修改
```

## 🎨 顏色和圖示系統

### 修復按鈕

- 🔧 **綠/藍漸層** = 安全、快速、技術
- 🤖 **粉/黃漸層** = 智能、創意、內容
- ⚠️ **橘紅漸層** = 強大、謹慎、進階
- ℹ️ **灰色** = 資訊、建議、手動

### 狀態指示

- ✅ **綠色** = 成功、完成、正常
- ⚠️ **黃色** = 警告、注意、重要
- ❌ **紅色** = 錯誤、失敗、禁止
- 🔵 **藍色** = 資訊、中性、一般

### 風險等級

- ⭐ **低** = 綠色框
- ⭐⭐ **中** = 黃色框
- ⭐⭐⭐ **高** = 紅色框
